# **App Name**: ChoripanFlow

## Core Features:

- POS Order Management: Cashiers can create new draft orders, easily add or remove menu items, and automatically recalculate order subtotal, tax, and total in real-time.
- Order Confirmation & Kitchen Handoff: Ability to confirm orders, updating their status to 'confirmed', and atomically generate a new kitchen ticket. Item prices are locked upon confirmation to prevent changes.
- Flexible Payment Processing: Enable cashiers to record multiple partial payments using various methods against a single order. The order is automatically marked as 'paid' once the sum of payments equals the total amount due.
- Real-time Sales Tracking: Upon successful order payment, the system automatically updates daily sales analytics, incrementing quantity and revenue for each purchased menu item.
- Kitchen Display System (KDS): Kitchen staff have a real-time interface to view incoming and pending kitchen tickets, allowing them to easily update ticket statuses (new, preparing, ready, served).
- Admin Sales Dashboard: Administrators can access a dashboard displaying key sales metrics, such as top-selling items by day, leveraging aggregated analytics data for business insights.

## Style Guidelines:

- Primary color: A warm, inviting ochre-orange (#B8732E), conveying comfort and quality without being overtly food-literal.
- Background color: A subtly warm, almost off-white (#F4F2F0) for a light, clean, and welcoming base, allowing content to stand out.
- Accent color: A vibrant, rich red (#DD3C3C), drawing attention to calls-to-action, warnings, and key status changes, analogous to the primary for cohesive warmth.
- All text: 'PT Sans' (sans-serif), offering a balanced, modern, and friendly readability for both transactional content and informative displays.
- Intuitive, clean, line-based icons that visually represent POS actions, menu categories, and order statuses, supporting efficient navigation.
- Modular and responsive layout design, optimized for efficient task completion on various devices (desktop for admin, tablets for POS/KDS), ensuring clear information hierarchy.
- Subtle and functional micro-interactions for feedback, such as item additions to the cart, payment confirmations, and status updates in the KDS, enhancing user experience without distraction.
